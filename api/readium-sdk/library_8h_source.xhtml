34 // Note that this is a library in the smallest sense: it keeps track of ePub files
35 // by their unique-identifier, storing the path to that ePub file *as given*. Its
36 // primary presence here is to allow for inter-publication linking. It is also
37 // designed to be subclassed in order to serve as the basis of a complete library
38 // in an application, with all the additional functionality that would involve.
39 //
40 // Note that each application would have a single library, accessible only through
41 // the MainLibrary() static method. A library can optionally be written out to
42 // disk and then loaded back in by calling MainLibrary() with a Locator instance
43 // at application startup. Once the singleton instance has been created,
44 // MainLibrary() will ignore its argument and always return that instance.
45 //
46 // Thoughts: OCF allows for multiple packages to be specified, but I don't see any
47 // handling of that in ePub3 CFI?
