Detailed Description

The ContentHandler class defines an interface by which resources of a particular media type can be handled.

Concrete subclasses exist which make use of a media handler defined in the OPF <bindings> element and for a custom renderer which takes a source path and a Package reference. The most common use is for the former case, when a resource with a foreign media-type is opened as a top-level document: the handler will instead open the DHTML handler document and pass that the source path as a URL query parameter.

A ContentHandler is initialized with a media-type string (similar to a MIME type) and a reference to the package to which this handler is assigned. It operates as a functor and is thus compatible with a C++11 std::function. Its 'call' operator takes two parameters, a Package-relative path and a list of key/value pairs defining any parameters (i.e. for a URL query string).

Remarks
The ContentHandler class keeps a non-owning reference to a Package instance.
